# AgriClaw Skill

## Input Rules

- Keep all prompts and replies in English.
- **Never ask for an input that the user has already provided in the current session.**
- For the recommendation workflow, collect missing inputs in this order:
  1. `location` and `crop` (ask together: e.g. `Please send: location crop`)
  2. `FLOCK_API_KEY` — **only ask if it has not already been set by the user**
     - accept `FLOCK_API_KEY=xxx` or plain key text
     - confirm receipt with: `FLOCK_API_KEY received` (never print the key value)
- For the **install workflow**, no inputs are required upfront — proceed immediately.

---

## Install / Update Workflow

When user says anything like "get the skill", "install", "update", or "set up AgriClaw":

1. Immediately run (no questions needed):
   ```
   bash install/install.sh
   bash install/install-skill.sh
   bash install/verify-skill-install.sh
   ```
2. On success, reply with exactly:
   `🌾 AgriClaw is ready, powered by Flock.io`
3. Do NOT show install paths or internal metadata unless explicitly asked.
4. Then ask for `location` and `crop` (and `FLOCK_API_KEY` if not already known) to run a first snapshot.
5. Run snapshot immediately once inputs are available:
   ```
   bash scripts/health-check.sh
   bash scripts/recommend.sh --location "<location>" --crop <crop>
   ```
6. Never claim success before `verify-skill-install.sh` passes.
7. If platform requires a new session to reload the skill, state it.

---

## Usage Workflow

When user asks for weather, prices, a recommendation, or a farm snapshot:

1. Ensure `location` and `crop` are known (ask only if missing).
2. Ensure `FLOCK_API_KEY` is set (ask only if not already provided this session).
3. Run:
   ```
   bash scripts/health-check.sh   # only when env is new/uncertain
   bash scripts/recommend.sh --location "<location>" --crop <crop>
   ```
4. Return output in this format:
   - a short status line if useful
   - `🌦 Weather` 1–2 lines
   - `💹 Prices` 1–3 bullet lines
   - `🦠 Pest Diagnosis & Policy` 2–4 bullet points
   - `✅ Recommended Actions` 2–3 bullet hints
   - `🤖 AI Recommendation (Flock.io)` 3–5 bullet points
   - `ℹ️ Heads-up` only when there is a real warning or missing optional config

The AI Recommendation section is generated by calling Flock.io using `FLOCK_API_KEY`.
If the key is absent, the section is skipped with a clear hint to the user.

---

## Model Switch Workflow

When user asks to switch model or provider:

1. Confirm target provider + model.
2. Run: `bash scripts/switch-model.sh --provider <provider> --model <model>`
3. Report: switched status, target config file, restart needed or not.

---

## Guardrails

- Never bypass the provider/model whitelist in `config/providers.json`.
- Show validation errors clearly if provider/model is not in the whitelist.
- Keep field-use output short and practical.
- Never ask for inputs the user has already given.
- Use a few clear emojis/icons in headings to improve readability, but keep body bullets plain and concise.
